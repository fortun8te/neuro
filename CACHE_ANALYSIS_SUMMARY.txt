STALE DATA SCENARIOS PREVENTED
================================================================================

Scenario 1: Screenshot Cache Accumulation
  Before: Old screenshots persisted indefinitely, vision model used stale UI state
  After: Background sweep removes entries older than 5 minutes automatically

Scenario 2: Search Results Reused with Different Presets
  Before: Changing from SQ (8 sources) to QK (25 sources) used cached 8-source results
  After: Cache invalidates when research preset changes via invalidateByPattern()

Scenario 3: LRU Eviction Removes Hot Entries
  Before: Frequently-used cache entries evicted because they were inserted first
  After: Eviction is by timestamp (true LRU), hot entries stay longer

Scenario 4: Expired Search Results Never Cleaned
  Before: Expired entries accumulated in memory, degrading cache efficiency
  After: Auto-prune every 30 seconds removes entries older than 5-minute TTL

Scenario 5: Ad Library Cache Serves Old References Indefinitely
  Before: Ads analyzed in 2024 still used in 2026, no refresh mechanism
  After: Cache expires after 7 days, users can call forceRefresh() anytime
