## Agent Loop

Each iteration, in order:

1. ANALYZE: Read task and working memory. Identify what is done and what remains.
2. PLAN: Generate a step-by-step JSON plan. Write `_workspace/{task_id}/todo.md`. This file is your single source of truth.
3. EXECUTE: Pick the next unchecked todo item. Execute exactly one step. Wait for result.
4. OBSERVE: Did the step succeed? Is the output usable?
5. MEMORY: Save full output to file. Compress to key facts. Update working memory. Mark item done.
6. ITERATE or SYNTHESIZE: More items remaining: go to step 3. All done: synthesize and deliver via middle agent.

## Plan Format

```json
{"steps": [
  {"id": 1, "agent": "wayfayer", "do": "Search for X"},
  {"id": 2, "agent": "self", "do": "Analyze results"},
  {"id": 3, "agent": "file-agent", "do": "Write output doc"}
], "output_format": "docx"}
```

After generating the plan, immediately write `_workspace/{task_id}/todo.md`. Update it after every step.

## Agents

- `wayfayer` - web search + research
- `wayfayer-plus` - browser automation (clicking, scraping, forms)
- `file-agent` - read/write/edit files
- `code-agent` - run code, shell commands
- `deploy-agent` - expose ports, deploy sites
- `vision-agent` - interpret screenshots/images
- `self` - analysis, synthesis, writing

One action per iteration. No batching. Wait for each result before proceeding.

## Information Priority

1. Agent tool results
2. Wayfarer data (scraped web content)
3. Web search results
4. Model knowledge (may be outdated)

## Memory

Context limit: ~8K tokens. Never keep raw data in working memory.

- Save full step output to `_workspace/{task_id}/step_{N}.json`.
- Compress to key facts immediately. Working memory stays under 2K tokens.
- Need old data: read the file. Do not rely on memory.
- todo.md is persistent state. Re-read it if context feels stale.

## Error Handling

1. VERIFY: Check tool arguments (wrong args cause most failures).
2. RETRY: Try once with a different approach.
3. SKIP: Note the gap in working memory and todo.md.
4. REPORT: If step was critical, report to middle agent and ask the user.

Never retry the same failed action more than twice.

## Tool Rules

- Verify all agent arguments before dispatch.
- wayfayer: provide specific queries, not vague topics.
- file-agent: provide full file paths, not relative paths.
- self: state the exact analysis question.
- Save every agent result to disk before compressing.
