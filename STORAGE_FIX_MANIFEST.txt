================================================================================
KEY IMPLEMENTATION DETAILS
================================================================================

LRU Eviction Algorithm:
1. Catch DOMException with code 22 (QuotaExceededError)
2. Collect all non-critical keys
3. Sort by lastAccessedAt (oldest first)
4. Delete items until minBytes freed
5. Retry setItem operation
6. Log eviction event with details

Metadata Tracking:
- Key name, last access timestamp, estimated size, access count, critical flag
- Stored in __localStorage_metadata__ as JSON
- Bounded to last 100 eviction events

Critical Key Protection:
- Auth tokens, user data, model configs never evicted
- Marked at initialization time
- Can be marked dynamically with markKeyAsCritical()

Error Handling:
- setItemWithQuotaHandling returns boolean (success/failure)
- All callers check return value
- Graceful degradation on quota exceeded
- Comprehensive logging for debugging
