2. frontend/services/loadMonitor.ts (407 lines)
   - LoadMonitor singleton managing load state
   - FEATURES:
     • Task recording/release with automatic cleanup
     • Per-model capacity checking (async)
     • Global concurrency enforcement (15 max)
     • FIFO queue with exponential backoff
     • GPU memory monitoring via Ollama health endpoint
     • Automatic queue processing when capacity freed
     • 5-minute max wait per task
     • 50-task queue limit
   
   - PUBLIC API:
     • checkAvailability(model): Non-blocking availability check
     • waitForCapacity(model, maxWaitMs): Block until capacity with queueing
     • recordTask(model): Mark task start, return taskId
     • releaseTask(model, taskId): Mark task complete
     • getModelLoad(model): Current load 0-100%
     • getGlobalLoad(): Global load 0-100%
     • getStatus(): Detailed per-model status object
     • getCliStatus(): Human-readable status string

3. frontend/services/modelFallback.ts (287 lines)
   - ModelFallback singleton implementing graceful degradation
   - FEATURES:
     • Automatic fallback to secondary model when primary unavailable
     • Exponential backoff retries (2s, 4s, 8s)
     • Cache fallback decisions for 1 minute
     • Logs model selection and reasons
     • Continues processing even if both models at capacity
   
   - PUBLIC API:
     • selectWithFallback(role, preferredModel): Intelligent selection
     • clearCache(): Reset fallback cache
     • getCacheStatus(): Debugging info

HOW TO USE
================================================================================

AUTOMATIC (recommended):
  // No code changes needed
  const response = await ollamaService.generateStream(
    'prompt', 'system', { model: 'qwen3.5:4b' }
  );
  // Load monitor handles queueing automatically

MANUAL CHECKS:
  const available = await loadMonitor.checkAvailability('qwen3.5:4b');
  if (available.available) { /* proceed */ }

CLI MONITORING:
  console.log(loadMonitor.getCliStatus());
  // Output: Active: 8/15 (53%) Qwen4b: 6/8, Qwen9b: 2/3

GRACEFUL DEGRADATION:
  const decision = await modelFallback.selectWithFallback(
    'research', 'qwen3.5:4b'
  );
  // Automatically falls back if qwen4b at capacity

FOR TESTING:
