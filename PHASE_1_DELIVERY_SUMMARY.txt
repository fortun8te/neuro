1. CIRCUIT BREAKER PATTERN
   File: src/utils/circuitBreaker.ts (170 lines)

   What: Fail-fast mechanism for unavailable services

   - Prevents cascading retry storms when services are down
   - Ollama: 5-failure threshold, 30s reset timeout
   - Wayfarer: 2-failure threshold, 15s reset timeout

   Integration:
   - ollama.ts: Circuit breaker before retry loop
   - wayfarer.ts: Circuit breaker on /research endpoint

   Impact: Prevents 6-30s latency on service failures


2. SCHEMA VALIDATION WITH ZOD
   Files:
   - src/utils/schemas/ollama.schemas.ts (80 lines)
   - src/utils/schemas/wayfarer.schemas.ts (95 lines)

   What: Runtime validation of API response structures
   - Validates every Ollama streaming JSON line
   - Validates Wayfarer /research and /screenshot responses
   - Lenient mode: logs validation failures but continues processing
   - Prevents malformed responses from crashing parser

   Schemas included:
   - Ollama: /api/generate, /api/chat, /api/tags
   - Wayfarer: /research, /screenshot/batch, /screenshot/single, sessions

   Impact: Prevents crashes from unexpected response formats


3. REQUEST QUEUE WITH RATE LIMITING
   File: src/utils/requestQueue.ts (150 lines)

   What: Enforces maximum concurrent requests per service
   - FIFO queue with configurable max concurrency
   - Prevents overwhelming remote services
   - Per-service concurrency limits
   - Dynamically adjustable concurrency

   Features:
   - Queue statistics: queued, active, completed, failed
   - Drain function: wait for all pending work to complete
   - Reset statistics for load testing

   Prepared for: SearXNG (10 req/min), Wayfarer (5 concurrent)

   Impact: Smooth throughput without rate limit errors


4. ERROR FORMATTER UTILITIES
   File: src/utils/errorFormatter.ts (180 lines)

   What: Standardized error message templates
   - Consistent format: "[Service] [Action] failed: [Reason] (HTTP [Status]). [Suggestion]"
   - Factory functions for common failure types
   - Human-readable suggestions for users

   Factories provided:
   - connectionFailed, httpError, timeout, invalidJson
   - schemaValidation, circuitBreakerOpen, rateLimited, aborted

   Impact: Better user experience and easier debugging
