================================================================================
CRITICAL FINDINGS (5 TOTAL)
================================================================================

1. NO CIRCUIT BREAKER [HIGH]
   - Service outages trigger cascading retries (6-12s latency per call)
   - Every failure = 3 retries before giving up
   - Recommendation: Implement circuit breaker (3 consecutive failures → fail-fast)
   - Effort: 2-3 hours
   - Files: /src/utils/ollama.ts, /src/utils/wayfarer.ts

2. SILENT WAYFARER FAILURES [MEDIUM]
   - research() returns empty result, not error
   - Callers must check result.pages.length === 0
   - No user notification of failure
   - Recommendation: Throw explicit "Wayfarer unavailable" error
   - Effort: 1 hour
   - File: /src/utils/wayfarer.ts

3. NO SCHEMA VALIDATION [MEDIUM]
   - JSON responses parsed without runtime type checking
   - Malformed responses can crash parser
   - No zod/ajv validation
   - Recommendation: Add schema validation for all API responses
   - Effort: 3-4 hours
   - Files: /src/utils/ollama.ts, /src/utils/wayfarer.ts

4. NO RATE LIMITING (SearXNG) [MEDIUM]
   - Config specifies searxngConcurrency: 32 but not enforced
   - No client-side request queue
   - Can hit 429 rate limits without intelligent backoff
   - Recommendation: Implement request queue with exponential backoff
   - Effort: 2-3 hours
   - File: /src/utils/wayfarer.ts

5. INCONSISTENT ERROR MESSAGES [LOW]
   - Some errors enhanced ("Is the server running?")
   - Others are raw HTTP status codes
   - Makes debugging harder, inconsistent UX
   - Recommendation: Standardize to [ServiceName] [Action] failed: [Reason]
   - Effort: 1-2 hours
   - Multiple files
