ROADMAP: 15 FEATURES ACROSS 3 PRIORITY TIERS
================================================================================

PRIORITY 1: CRITICAL (Week 1-5) — Core gaps that block GAIA improvement
─────────────────────────────────────────────────────────────────────────────

🔴 P1-A: Query-Level Caching & Memoization
   Impact: SPEED (5-10% faster on repeated queries)
   Effort: 3 days | Feasibility: HIGH
   Why: GAIA has repeated questions. Hash-based LLM cache saves 30% of model calls.
   Action: Add cache layer to ollama.ts, TTL 24h, per-model + per-prompt

🔴 P1-B: Speculative Decoding
   Impact: SPEED (2-3x faster generation)
   Effort: 4 days | Feasibility: MEDIUM
   Why: Token generation is bottleneck. Small model generates draft, large validates.
   Action: Implement dual-model orchestration (qwen2b → qwen9b validation)

🔴 P1-C: Long-Context Management (100K+ tokens)
   Impact: CAPABILITY (unblock document analysis)
   Effort: 2.5 days | Feasibility: HIGH
   Why: Can't analyze large documents/codebases. Need hierarchical chunking.
   Action: Use Context-1 semantic chunking + reranking + sliding window

─────────────────────────────────────────────────────────────────────────────

PRIORITY 2: HIGH-VALUE (Week 5-7) — Leverage + quality improvements
─────────────────────────────────────────────────────────────────────────────

🟠 P2-A: Quality Metrics & Automated Evaluation
   Impact: QUALITY (gate bad answers, auto-retry)
   Effort: 3 days | Feasibility: HIGH
   Why: No feedback loop today. Eval model scores output; if <60, retry automatically.
   Action: Implement 5-criteria eval (coherence, relevance, completeness, accuracy,
           specificity). Route to larger model or adjusted prompt if needed.

🟠 P2-B: Autonomous Improvement Loop
   Impact: LEARNING (fewer repeated mistakes)
   Effort: 4 days | Feasibility: MEDIUM
   Why: Same failures repeat. LLM analyzes root cause, adjusts strategy, retries.
   Action: Add failure tracking + root cause analysis + adaptation strategy routing

🟠 P2-C: Knowledge Base / RAG System
   Impact: REUSE (20%+ faster on follow-ups)
   Effort: 4 days | Feasibility: MEDIUM
   Why: All findings trapped in IndexedDB. Chroma + embeddings enable retrieval.
   Action: Index all findings in Chroma. On new question, retrieve similar past answers.
           Surface "Similar to Q3: here's what we learned" in UI.

🟠 P2-D: Vision & Multimodal Integration
   Impact: CAPABILITY (true image understanding)
   Effort: 5-6 days | Feasibility: MEDIUM
   Why: Wayfarer Plus isolated. Need qwen-vl in research, make, test stages.
   Action: Integrate vision model, add image analysis to research, visual reasoning to make

─────────────────────────────────────────────────────────────────────────────

PRIORITY 3: NICE-TO-HAVE (Weeks 8+) — Usability & team features
─────────────────────────────────────────────────────────────────────────────

🟡 P3-A: Advanced Search & Full-Text Retrieval
   Impact: DISCOVERY (find past work quickly)
   Effort: 3 days | Feasibility: HIGH
   Why: Can't search cycles. Need FTS + semantic search across all findings.
   Action: SQLite FTS + embeddings, search UI with facets (date, stage, quality)

🟡 P3-B: Export & Sharing (PDF, HTML, JSON, PowerPoint)
   Impact: SHARING (stakeholder visibility)
   Effort: 4-5 days | Feasibility: HIGH
   Why: Work trapped in NEURO. Stakeholders can't review offline.
   Action: Multiple export templates (PDF professional, HTML interactive, JSON data,
           Markdown wiki, PowerPoint slides)

🟡 P3-C: Real-Time Collaboration & Multi-User
   Impact: TEAMWORK (async + sync collaboration)
   Effort: 5-7 days | Feasibility: MEDIUM-HIGH
   Why: Single-user only. Teams can't work together.
   Action: WebSocket server, state sync, conflict resolution, role-based access

🟡 P3-D: Custom Workflow Designer
   Impact: FLEXIBILITY (adapt to different question types)
   Effort: 6-8 days | Feasibility: MEDIUM
   Why: 8-stage pipeline fixed. Some questions need less research, more ideation.
   Action: Visual workflow builder (React Flow), DAG execution, conditional routing

🟡 P3-E: Model Adaptation & Few-Shot Learning
   Impact: QUALITY (better outputs with in-context examples)
   Effort: 3-4 days | Feasibility: HIGH
   Why: Static prompts. Few-shot examples from past successes improve quality.
   Action: Few-shot library per task type, retrieve good examples, append to prompt

🟡 P3-F: Cost Tracking & Optimization
   Impact: BUSINESS (visibility, optimization)
   Effort: 3-4 days | Feasibility: HIGH
   Why: No per-query cost breakdown. Can't optimize spending.
   Action: Fine-grained token tracking per stage, dashboard, auto-suggest smaller models

🟡 P3-G: Automated Testing Framework
   Impact: STABILITY (regression detection)
   Effort: 4-5 days | Feasibility: MEDIUM
   Why: No regression testing. Optimizations might silently break things.
   Action: Test suite (30-50 questions), baseline comparison, CI/CD integration

🟡 P3-H: Mobile Responsiveness
   Impact: USABILITY (view on mobile)
   Effort: 3-4 days | Feasibility: MEDIUM
   Why: Desktop-first UI. Can't view results on phone.
   Action: Responsive design, mobile-friendly streaming

RISK ASSESSMENT
================================================================================

Technical Risks:
  • Caching breaks with model updates → VERSION PROMPTS
  • Spec decoding quality drop → VALIDATE FIRST, EASY ROLLBACK
  • Long-context OOM → ADD MEMORY LIMITS
  • Chroma diverges → REGULAR REINDEX
  • Eval wrong → VALIDATE ON TEST SET FIRST
  • Retry loops → LOOP DETECTION + MAX COUNT

Mitigation:
  ✓ Phased rollout (feature flags)
  ✓ A/B testing (old vs new)
  ✓ Audit trail (enable debugging)
  ✓ Rollback plan (per feature)
  ✓ Test suite (regression detection)
